## 1. What shipped

### v1.9.0 — cipher detail + SSL Labs
- **Full TLS cipher detail in the UI.** The complete active cipher
  enumeration existed in `domain_extra['cipher_enum']` but never reached the
  dashboard (the modal truncated to 2 ciphers). Now: the assessor merges the
  full enumerated suite set (IANA names) into `assessments.cipher_suites`;
  `/api/domain/<domain>` returns the latest `cipher_enum`/`chain`/`cdn`/
  `ssllabs` blobs; the modal shows a per-security-level summary with a
  **Full TLS Details** drill-down view (`view-domain-full`) listing every
  accepted suite (protocol, bits, category, colour-coded assessment).
- **CIPHER_ENUM findings now name the suites** to remove (message + a
  machine-readable `ciphers` field on each finding).
- **Passive cipher normalisation** OpenSSL→IANA before `_assess_cipher()` —
  fixed a latent bug where guideline lists (IANA) never matched passive
  OpenSSL names and always fell through to a generic score.
- **CAMELLIA / SEED probes** added to `cipher_enum.py` (coverage gap vs SSL
  Labs on European servers).
- **SSL Labs API v4 integration** (`scanner/ssllabs_client.py`): cache-only
  during scan runs (`fromCache=on`, never triggers external assessments
  inline); on-demand fresh assessment from the detail view (`startNew=on`,
  `publish=off`, `scan.run` permission), polled by the UI. Grade + report
  link shown in the modal/detail. **Display only — does not affect the PQC
  score** (explicit decision). Requires one-time `register_email()` with an
  organisational email; config `ssllabs.email` or `PQC_SSLLABS_EMAIL`.
- **`scripts/reassess_all.py`**: reassess every existing domain. Score-only
  by default (no traffic/CPU-light, reuses stored blobs, regenerates named
  findings); `--rescan` for a resource-guarded network rescan
  (`--workers`/`--sleep`/`--limit`/`--only-missing`/`--dry-run`).

### v1.9.1 — MX + SMTP/STARTTLS + scoring
- **MX host normalisation.** MX rdata `"<priority> <exchange>"` leaked
  verbatim (`"5 SMTP.domain.com"`) via the DNSDumpster and passive paths.
  `_normalise_mx_host()` strips the priority, lower-cases, drops the trailing
  dot, and rejects non-hostnames; applied at every ingest point.
- **SMTP/STARTTLS reported "no TLS" on 465/587/2525.** `probe_starttls()`
  hardcoded `if port in (25, 587)`, so other SMTP ports did a *direct* wrap
  on a plaintext socket and failed. Now dispatches by **protocol family**;
  added **2525**; unknown ports fail explicitly instead of masquerading as
  no-TLS; hardened the SMTP greeting/EHLO parse; `scanning.use_starttls` is
  now actually read from config.
- **No-TLS domains scored "30 weak" instead of "na".** `assess_domain()`
  only returned `na` for an *empty* scan list; a host whose every scan
  *failed* fell through and was scored purely on the PQC penalty (`+30`).
  Now returns `na` (score 0, no findings) when no service handshakes.
  `reassess_all.py` writes a fresh `na` row for such domains so the newest
  row is always correct.
- **`scripts/fix_mx_entries.py`**: repairs malformed MX values. Repairs BOTH
  the `domain` primary-key column across all seven domain-keyed tables
  (rename / merge-on-collision / drop unrecoverable) AND the `dns_enum`
  blobs. No network, idempotent, `--dry-run`.

## 2. Production actions taken this session (on the live DB)

1. Ran `fix_mx_entries.py` — repaired malformed `domain` keys
   (`5 smtp.bde.es` → `smtp.bde.es`, `20 mail01.bancaditalia.it` → …,
   `primary DNS domain` deleted).
2. Discovered a batch of no-TLS hosts (`a-bancox2.bde.es`, `x-www.bde.es`,
   several `escb.eu`) showing **"30 weak"**. Root cause: a reassess run
   (`06b6e9c1`, 2026-07-10 08:53) executed with the **pre-fix assessor**.
   Confirmed via `services_assessed=0` on those rows.
3. Deleted the erroneous rows manually:
   `DELETE FROM assessments WHERE services_assessed=0 AND level!='na';`
   After the delete, the hosts correctly show **No TLS** again (the older
   `na` row became the newest).

The fixed assessor + reassess prevent recurrence: a re-run now writes a
correct `na` that becomes the newest row.
